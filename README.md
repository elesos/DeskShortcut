# DeskShortcut


Desktop + Shortcut 是“桌面快捷方式管理工具”，不偏启动器、不偏美化软件，适合这个产品定位。


DeskShortcut 桌面快捷方式管理软件 PRD

1. 产品概述

1.1 产品名称

DeskShortcut

名称来源：

* Desk：来自 Desktop，代表桌面
* Shortcut：快捷方式
* 合起来表达：桌面快捷方式管理工具

1.2 产品定位

DeskShortcut 是一款基于 Tauri 技术开发的轻量级桌面快捷方式管理软件，主要用于集中收纳和管理 Windows 桌面快捷方式。

用户可以将桌面快捷方式拖入 DeskShortcut，软件自动读取快捷方式信息，包括软件名称、图标、目标路径、启动参数、工作目录等。导入成功后，用户可以选择自动删除桌面上的原快捷方式。之后用户无需再从桌面启动软件，而是直接在 DeskShortcut 中点击图标启动。

产品核心目标是让用户桌面更干净，同时保留原快捷方式的完整启动能力。

⸻

2. 产品背景

很多 Windows 用户桌面上长期堆积大量软件快捷方式，导致：

* 桌面混乱
* 软件查找效率低
* 快捷方式不好分类
* 常用软件和低频软件混在一起
* 删除桌面图标时又担心影响软件启动

现有启动器类软件功能较多，容易变成桌面美化、插件平台或全局搜索工具。DeskShortcut 不做复杂扩展，专注一个核心场景：

把桌面快捷方式收进软件里，桌面变干净，以后从软件里启动。

⸻

3. 目标用户

3.1 普通电脑用户

* 桌面上有较多软件快捷方式
* 希望桌面保持整洁
* 希望快速找到常用软件
* 不想配置复杂启动器

3.2 办公用户

* 经常使用固定办公软件
* 希望按工作用途分组
* 希望减少桌面图标数量
* 需要稳定、轻量、易用的软件入口

3.3 工具型用户

* 软件数量较多
* 需要保留快捷方式启动参数
* 需要管理特殊启动方式的软件
* 关注目标路径、工作目录、启动参数等信息

3. 支持导入后删除桌面快捷方式

5. 核心使用场景

5.1 导入桌面快捷方式

用户桌面上有很多软件快捷方式，希望集中收纳。

流程：

1. 用户打开 DeskShortcut
2. 将桌面 .lnk 快捷方式拖入软件窗口
3. DeskShortcut 自动解析快捷方式
4. 显示导入确认窗口
5. 用户选择所属分组，默认有一个分组，用户可以在此处新建分组
6. 用户确认是否删除桌面原快捷方式
7. 软件保存快捷方式信息
8. 若用户确认删除，则删除桌面上的 .lnk 文件
9. 快捷方式显示在 DeskShortcut 中

⸻

5.2 从 DeskShortcut 启动软件

用户不再从桌面点击软件，而是从 DeskShortcut 中启动。

流程：

1. 用户打开或呼出 DeskShortcut
2. 选择对应分组
3. 点击软件图标
4. DeskShortcut 按保存的启动配置启动软件
5. 启动配置包括目标路径、启动参数、工作目录

⸻

5.3 分组管理软件

用户可以按用途管理快捷方式。

示例分组：

* 常用
* 办公
* 开发
* 设计
* 游戏
* 系统工具
* 未分组

用户可以在不同分组之间切换，快速找到对应软件。

⸻

5.4 靠边隐藏

用户希望 DeskShortcut 日常不占用屏幕空间。

流程：

1. 用户将窗口拖到屏幕顶部
2. DeskShortcut 自动吸附到边缘
3. 鼠标离开后窗口自动隐藏
4. 鼠标移动到屏幕边缘时窗口自动显示
5. 用户点击软件启动

⸻

6. 功能需求

6.1 快捷方式导入

6.1.1 拖拽导入

DeskShortcut 支持用户将 Windows 快捷方式文件拖入软件。

首版支持：

* .lnk 文件

首版暂不支持：

* macOS alias
* Linux .desktop
* 浏览器 URL 快捷方式
* 文件夹批量扫描

6.1.2 快捷方式信息读取

导入 .lnk 后，需要读取以下信息：

字段	说明	是否必需
快捷方式名称	用于软件内展示	是
原快捷方式路径	原 .lnk 文件位置	是
目标路径	实际启动程序路径	是
启动参数	.lnk 中的 arguments	是
工作目录	Start in / Working Directory	是
图标路径	快捷方式图标来源	是
图标索引	icon index	否
描述	快捷方式描述	否
快捷键	Windows 快捷方式热键	否
显示方式	正常窗口、最大化、最小化	否

重点要求：

DeskShortcut 不能只保存 exe 路径，必须保存启动参数和工作目录，否则部分软件无法按原快捷方式方式启动。

⸻

6.2 导入确认

用户拖入快捷方式后，弹出确认窗口。

6.2.1 确认窗口展示内容

展示：

* 图标
* 名称
* 目标路径
* 启动参数
* 工作目录
* 所属分组
* 是否删除桌面快捷方式

6.2.2 默认选项

默认勾选：

导入后删除桌面快捷方式

说明文案：

仅删除桌面上的快捷方式，不会删除软件本体。

按钮：

* 取消
* 确认导入

⸻

6.3 自动删除桌面快捷方式

6.3.1 删除规则

用户确认导入并勾选“导入后删除桌面快捷方式”后，DeskShortcut 删除原 .lnk 文件。

删除要求：

* 只删除用户拖入的 .lnk 文件
* 不删除目标程序
* 不删除目标程序所在文件夹
* 不递归删除任何目录
* 删除前校验文件扩展名
* 删除失败不影响已导入数据

6.3.2 删除失败处理

场景	处理
文件不存在	提示“原快捷方式已不存在”
权限不足	提示“权限不足，无法删除快捷方式”
文件被占用	提示“快捷方式正在被占用，请稍后手动删除”
非 .lnk 文件	不允许删除
删除失败	保留导入记录，并提示用户

6.3.3 安全文案

删除前需要明确提示：

DeskShortcut 只会删除桌面快捷方式，不会删除软件本体。

⸻

6.4 软件启动

6.4.1 点击启动

用户点击 DeskShortcut 中的软件图标后，软件按照保存的快捷方式配置启动目标程序。

启动时使用：

* 目标路径
* 启动参数
* 工作目录

6.4.2 启动参数支持

必须支持读取、保存、编辑和传递启动参数。

示例：

目标路径：
C:\Program Files\App\app.exe
启动参数：
--profile user1 --safe-mode
工作目录：
C:\Program Files\App

启动时应等价于用户点击原 Windows 快捷方式。

6.4.3 工作目录支持

启动程序时需要设置 working directory。

如果工作目录不存在：

1. 提示用户工作目录不存在
2. 可尝试不带工作目录启动
3. 允许用户进入编辑页修正

6.4.4 启动失败处理

失败原因	提示
目标程序不存在	目标程序不存在，可能已被移动或卸载
工作目录不存在	工作目录不存在，请检查快捷方式配置
权限不足	启动失败，可能需要管理员权限
启动参数异常	启动参数可能有误，请检查配置
未知错误	启动失败，请查看错误信息

⸻

6.6 分组管理

6.6.1 默认分组

系统默认分组：

* 全部
* 常用
* 未分组

说明：

* “全部”展示所有快捷方式
* “常用”由用户手动添加
* “未分组”用于存放未指定分组的快捷方式

6.6.2 分组操作

支持：

* 新建分组
* 重命名分组
* 删除分组
* 调整分组顺序
* 将快捷方式移动到其他分组

6.6.3 删除分组规则

删除分组时，不删除分组内快捷方式。

处理方式：

删除分组后，该分组内的快捷方式自动移动到“未分组”。

6.6.4 快捷方式排序

支持在分组内调整快捷方式顺序。

首版可支持：

* 右键菜单上移 / 下移
* 编辑排序值

后续增强：

* 拖拽排序

6.7 靠边隐藏

6.7.1 窗口吸附

用户将 DeskShortcut 拖动到屏幕边缘时，窗口自动吸附。

首版支持：

* 顶部

首版暂不重点支持：

* 底部，左侧和右侧

原因：

* 底部容易和 Windows 任务栏冲突

6.7.2 自动隐藏

窗口处于吸附状态时，鼠标离开窗口后自动隐藏。

隐藏规则：
* 顶部吸附：窗口向上隐藏

隐藏后保留 2-5 像素触发区域。

6.7.3 鼠标触边显示

当鼠标移动到触发区域时，窗口自动显示。

建议参数：

项目	建议值
显示延迟	0-200ms
隐藏延迟	500-1000ms
触发区域	2-5px

6.7.4 固定窗口

窗口提供固定按钮。

状态：

* 固定：窗口不自动隐藏
* 自动隐藏：鼠标离开后隐藏

10. 技术方案

10.1 技术栈

推荐技术栈：

Tauri + Vue 3 + TypeScript + Rust

也可以选择：

Tauri + React + TypeScript + Rust

如果项目偏桌面工具，推荐 Vue 3，开发效率较高，界面状态也相对简单。

⸻

10.2 Tauri 负责能力

Tauri 前端负责：

* 界面展示
* 拖拽交互
* 分组管理
* 设置页
* 快捷方式列表管理

Rust 后端负责：

* 解析 .lnk
* 删除快捷方式文件
* 启动目标程序
* 读取图标
* 保存本地配置
* 控制窗口位置
* 靠边隐藏相关窗口行为

⸻

10.3 快捷方式解析

Windows .lnk 文件需要调用系统接口解析，不能按普通文本读取。

建议使用：

* Windows COM
* IShellLinkW
* IPersistFile

需要封装 Tauri command：

parse_shortcut(path: string) -> ShortcutInfo

返回结构：

{
  "name": "Visual Studio Code",
  "targetPath": "C:\\Users\\User\\AppData\\Local\\Programs\\Microsoft VS Code\\Code.exe",
  "arguments": "--reuse-window",
  "workingDirectory": "C:\\Users\\User\\AppData\\Local\\Programs\\Microsoft VS Code",
  "iconPath": "C:\\Users\\User\\AppData\\Local\\Programs\\Microsoft VS Code\\Code.exe",
  "iconIndex": 0
}

⸻

10.4 启动程序

启动程序时必须带上完整启动信息。

Tauri command：

launch_shortcut(shortcutId: string)

启动逻辑：

1. 根据 shortcutId 获取快捷方式配置
2. 检查 targetPath 是否存在
3. 检查 workingDirectory 是否存在
4. 传递 arguments
5. 按配置启动程序
6. 返回启动结果

⸻

10.5 删除快捷方式

Tauri command：

delete_shortcut_file(path: string)

删除前校验：

* 文件存在
* 文件后缀为 .lnk
* 文件路径等于导入时保存的 originalShortcutPath
* 不能是 targetPath
* 不能是目录

⸻

10.6 本地存储

首版建议使用本地 JSON 文件。

原因：

* 数据量小
* 结构简单
* 易开发
* 易导入导出
* 易备份

后续如功能复杂，可升级 SQLite。

保存内容：

* 快捷方式列表
* 分组列表
* 用户设置
* 窗口位置
* 图标缓存索引

⸻

10.7 图标处理

图标来源优先级：

1. 快捷方式指定图标
2. 目标程序 exe 图标
3. 默认应用图标

建议缓存图标，避免每次启动软件都重新读取。

缓存字段：

* shortcutId
* iconPath
* iconIndex
* cachedIconPath

        ↓

        ↓

        ↓
展示导入确认窗口
        ↓
用户选择分组
        ↓
用户确认是否删除桌面快捷方式
        ↓
保存快捷方式数据
        ↓
如勾选删除，则删除原 .lnk
        ↓
导入完成

        ↓
读取保存的快捷方式配置
        ↓

        ↓
按 targetPath + arguments + workingDirectory 启动
        ↓
返回启动成功或失败

⸻

12.3 靠边隐藏流程

用户拖动窗口到屏幕边缘
        ↓

13. 异常处理

13.1 导入异常

异常	处理
非 .lnk 文件	提示“当前仅支持 Windows 快捷方式”
快捷方式解析失败	提示“快捷方式读取失败，可手动添加”
目标路径不存在	允许导入，但标记异常
没有权限读取	提示权限不足
重复导入	提示已存在，可覆盖或取消

13.2 删除异常

异常	处理
原快捷方式不存在	提示已不存在
权限不足	提示无法删除
文件被占用	提示稍后手动删除
非快捷方式文件	不执行删除
删除失败	保留导入数据

13.3 启动异常

异常	处理
目标程序不存在	提示用户编辑目标路径
工作目录不存在	尝试不带工作目录启动
权限不足	提示可能需要管理员权限
参数异常	提示检查启动参数

14. 重复导入规则

重复判断优先级：

1. originalShortcutPath 相同
2. targetPath + arguments 相同
3. 名称相同但路径不同

处理方式：

情况	处理
完全相同	提示“已存在”
路径相同但参数不同	允许作为新快捷方式导入
名称相同但路径不同	允许导入，名称自动加后缀
用户选择覆盖	更新已有快捷方式

16. 关键风险

16.1 快捷方式解析风险

风险：

Windows .lnk 文件解析需要调用系统接口，不能简单读取文件内容。

解决方案：

* 使用 Rust 调用 Windows COM
* 使用 IShellLinkW
* 使用 IPersistFile
* 解析失败时允许用户手动编辑

16.2 误删风险

风险：

用户可能担心删除桌面快捷方式会删除软件本体。

解决方案：

* 删除前明确提示
* 只允许删除 .lnk
* 不允许删除 exe
* 不允许删除文件夹
* 删除失败不影响导入结果

16.3 启动参数丢失风险

风险：

如果只保存目标 exe，不保存参数，会导致部分软件启动方式和原快捷方式不一致。

解决方案：

* 导入时必须读取 arguments
* 数据结构必须保存 arguments
* 编辑页必须展示 arguments
* 启动时必须传递 arguments

16.4 靠边隐藏兼容风险

风险：

不同分辨率、多显示器、任务栏位置可能影响靠边隐藏。

解决方案：

* 第一版优先支持主显示器
* 第一版支持左、右、上三个方向
* 后续再增强多显示器和任务栏避让

18. 产品一句话

DeskShortcut 是一个轻量级桌面快捷方式管理工具，用户把桌面快捷方式拖进来，软件自动读取启动路径、参数和图标，并可删除桌面原快捷方式，以后直接从 DeskShortcut 分组启动软件。
